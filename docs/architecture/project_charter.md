# 1. Purpose

Project **GnSyS** is a long-term Oracle Cloud Infrastructure (OCI) engineering initiative focused on designing, implementing, documenting, and continuously improving an enterprise-style cloud environment.

The project prioritizes engineering discipline, architectural thinking, documentation quality, and practical cloud experience over simply completing tutorials or passing certification exams.

---

# 2. Vision

To build a production-inspired Oracle Cloud Infrastructure environment that demonstrates enterprise architecture principles, cloud engineering best practices, and continuous technical growth.

The repository is intended to serve both as a personal learning platform and as a professional engineering portfolio.

---

# 3. Mission

Design, deploy, document, and maintain a complete Oracle Cloud Infrastructure environment by applying structured engineering practices while developing the knowledge and experience required for professional Oracle Cloud certifications.

---

# 4. Strategic Objectives

Project GnSyS pursues the following objectives:

- Learn Oracle Cloud Infrastructure through hands-on implementation.
- Develop enterprise cloud architecture skills.
- Apply consistent engineering standards.
- Document every architectural decision.
- Build reusable deployment patterns.
- Prepare for Oracle OCI certifications.
- Publish a portfolio-quality GitHub repository.
- Continuously evolve the platform as new OCI technologies are introduced.

---

# 5. Project Scope

The initial project scope includes the following Oracle Cloud Infrastructure services:

- Identity and Access Management (IAM)
- Compartments
- Virtual Cloud Networks (VCN)
- Routing
- Internet Gateway
- NAT Gateway
- Service Gateway
- Security Lists
- Network Security Groups
- Compute Instances
- Block Volumes
- Object Storage
- Autonomous Database
- Monitoring
- Logging
- Notifications
- Budgets
- Cost Analysis

Future phases are expected to include:

- Terraform
- OCI DevOps
- Oracle Kubernetes Engine (OKE)
- Bastion Service
- Load Balancing
- Web Application Firewall (WAF)
- High Availability
- Disaster Recovery
- Multi-Region Architectures
- Infrastructure as Code (IaC)

---

# 6. Out of Scope

The following topics are intentionally excluded from the current project scope:

- Commercial production workloads
- Large-scale enterprise deployments
- Multi-cloud architectures
- Oracle SaaS products
- Custom application development

These subjects may become part of future project phases.

---

# 7. Guiding Principles

Project GnSyS follows the engineering principles below.

## Documentation First

Every implementation must be documented.

---

## Design Before Deployment

Every deployed resource must have a clearly defined purpose.

---

## Simplicity

Prefer simple, maintainable, and understandable architectures.

---

## Consistency

Apply naming standards, tagging standards, and architectural conventions consistently across the project.

---

## Security by Default

Security must be considered throughout every implementation.

---

## Reproducibility

Every laboratory should be reproducible from scratch.

---

## Continuous Improvement

The project should continuously evolve as new OCI services and architectural patterns are learned.

---
